## Component Inventory (Phase-Specific)

### Lock Badge (new pattern)

Small Lock icon overlaid on existing action buttons to indicate auth-required.

**Variant A -- Icon button overlay (SpotDetail header favorite, share-like buttons):**
- Lock icon: `Lock` from lucide-react, size 10px
- Position: absolute, top -2px, right -2px
- Color: Slate 400 (#94a3b8)
- Background: White circle, 16px diameter, border 1px Slate 200
- The parent button adds `position: relative` and `overflow: visible`

**Variant B -- Inline suffix on text buttons (review CTA, session join):**
- Lock icon: `Lock` from lucide-react, size 14px
- Position: inline, after button label text, gap 4px
- Color: Slate 400 (#94a3b8)
- No background circle

**Variant C -- Inline suffix on NavBar tab (Favorites tab):**
- Lock icon: `Lock` from lucide-react, size 12px
- Position: inline, after tab label, gap 4px
- Color: Slate 400 (#94a3b8) when tab is inactive; Sky 500 when tab is active/pressed
- No background circle

### Anonymous Profile Screen (new component branch in Profile.tsx)

Full-height centered layout within the Profile tab area.

- **Container**: flex column, items-center, justify-center, full height, px-24px, bg-slate-50
- **Illustration area**: 120px x 120px circle, bg-sky-50, centered, with a `MapPin` icon (lucide, size 48, color Sky 500) -- reuses existing project iconography
- **Title**: "Rejoins la communaute Updock" (fr) / "Join the Updock community" (en) -- text-xl (20px), font-bold (700), text-slate-800, text-center, mt-24px
- **Subtitle**: "Decouvre les spots, partage tes sessions et retrouve la communaute." (fr) / "Discover spots, share your sessions and join the community." (en) -- text-base (16px), font-normal (400), text-slate-500, text-center, mt-8px, max-w-280px
- **Primary button "Se connecter"**: full width (max-w-320px), bg-sky-500, text-white, font-bold, py-16px, rounded-xl, mt-32px -- matches existing AuthModal button style
- **Secondary button "S'inscrire"**: full width (max-w-320px), bg-white, border border-slate-200, text-slate-700, font-bold, py-16px, rounded-xl, mt-12px
- **Settings section**: Language toggle row remains visible below the auth CTA area, separated by 48px margin-top. Same style as existing Profile settings rows.

### Auth-Gated Action Pattern (behavior, not a new component)

Every protected action button follows this contract:
1. Button is always rendered (never hidden for anonymous users)
2. If `!user`: display Lock badge variant (A, B, or C depending on button type)
3. If `!user`: onClick calls `onOpenAuth()` instead of the original action
4. If `user`: render exactly as today (no lock badge, original onClick)

Protected actions with their lock variant:
| Action | Component | Lock Variant |
|--------|-----------|-------------|
| Toggle favorite (SpotDetail header) | SpotDetail.tsx | A (icon overlay) |
| Leave a review | SpotDetail.tsx (reviews tab) | B (inline suffix) |
| Join/create session | SpotDetail.tsx (sessions tab) | B (inline suffix) |
| Favorites tab | NavBar.tsx | C (NavBar suffix) |
| Add spot button | Map.tsx / NavBar.tsx | A (icon overlay) |

## Interaction States

### Anonymous user taps a protected button
1. Button shows Lock badge (variant A, B, or C)
2. On tap: AuthModal opens with default sign-in view
3. On successful auth: AuthModal closes, user state updates, lock badges disappear reactively
4. On AuthModal close (without auth): nothing changes, user stays on current screen

### Anonymous user taps Favorites tab
1. NavBar Favorites tab shows Lock badge (variant C)
2. On tap: AuthModal opens (not a tab switch)
3. After auth: Favorites tab becomes functional, user is switched to Favorites tab

### Anonymous user views Profile tab
1. Profile tab renders the anonymous profile screen (not the authenticated profile)
2. "Se connecter" button opens AuthModal in sign-in mode
3. "Creer un compte" button opens AuthModal in sign-up mode
4. After successful auth: Profile tab re-renders with the authenticated profile (no redirect)

### Post-login behavior (Claude's discretion resolution)
- After login from any AuthModal trigger: close the modal, user stays on current screen/tab
- No automatic replay of the gated action -- user re-taps the button (now ungated)
- Exception: login from Profile tab keeps user on Profile tab (CONTEXT.md locked decision)

## Animation

- AuthModal open/close: use existing framer-motion animation (already implemented)
- Lock badge: no animation (static icon, appears/disappears reactively based on auth state)
- Anonymous profile screen: subtle fade-in on mount using framer-motion `initial={{ opacity: 0 }} animate={{ opacity: 1 }}` matching existing LandingPage pattern

---

## Accessibility

- Lock badge icons include `aria-label="Connexion requise"` (fr) / `"Sign in required"` (en)
- All protected buttons remain focusable and keyboard-accessible
- Anonymous profile screen buttons have sufficient color contrast (Sky 500 on white text passes WCAG AA)
- Touch targets: all buttons maintain minimum 44px touch target (existing w-10 h-10 = 40px icon buttons are padded by parent spacing to reach 44px effective target)
