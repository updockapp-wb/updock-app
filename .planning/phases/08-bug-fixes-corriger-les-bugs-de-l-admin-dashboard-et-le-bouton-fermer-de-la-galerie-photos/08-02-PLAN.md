**D-01 — Fix X button not closing dashboard:**

The X button on line 52 has `onClick={onClose}` which should work. The likely issue is that the edit overlay (`editingSpot`) uses `className="absolute inset-0 bg-white z-50"` (line 177) which covers the entire content area including potentially intercepting pointer events. However the X button is in the header which is OUTSIDE the `relative` content div. Let me trace the DOM:

- The outer fixed div (`fixed inset-0 z-[5000]`) contains a flex-col:
  - Header div (with X button) — NOT inside the relative container
  - Content div (`flex-1 overflow-y-auto relative`) — contains the edit overlay with `absolute inset-0 z-50`

The edit overlay is `absolute inset-0` inside the content div which has `relative`. So it should NOT cover the header. But wait — the content div does NOT have `position: relative` explicitly. It has `className="flex-1 overflow-y-auto p-6 bg-slate-50 relative"` — yes it does have `relative`.

So the header X should be clickable. The real issue could be:
1. On mobile, touch events on the X button may be swallowed by the AnimatePresence/motion wrapper
2. The `onClose` function might not trigger a re-render that causes AnimatePresence exit

**Fix approach:** Add `e.stopPropagation()` to the X button click handler to prevent event bubbling. Also add explicit `pointer-events-auto` and a larger touch target. Replace line 52:

```tsx
<button
  onClick={(e) => { e.stopPropagation(); onClose(); }}
  className="p-3 bg-white/10 rounded-full hover:bg-white/20 transition-colors active:bg-white/30"
  aria-label="Close"
>
  <X size={20} />
</button>
```

Change padding from `p-2` to `p-3` for a bigger touch target on mobile.

**D-05 — Lightbox X button not responding on mobile:**

The lightbox (lines 695-752) is portalled to `document.body` via `createPortal`. The X button on line 706 has `onClick={() => setIsImageOpen(false)}` but lacks `e.stopPropagation()`.

The parent `motion.div` (line 703) also has `onClick={() => setIsImageOpen(false)}` as a backdrop dismiss. On mobile, touch events can behave unexpectedly when both parent and child have click handlers — especially inside a portal on iOS Capacitor.

**Fixes:**

1. **Add `e.stopPropagation()` to the X button** (line 706):
   ```tsx
   <button
     onClick={(e) => { e.stopPropagation(); setIsImageOpen(false); }}
     className="absolute top-6 right-6 p-4 bg-white/10 hover:bg-white/20 rounded-full transition-colors z-10"
   >
     <X size={24} className="text-white" />
   </button>
   ```

2. **Add `onTouchEnd` handler as fallback** for iOS Capacitor where `onClick` on overlays may not fire reliably:
   ```tsx
   <button
     onClick={(e) => { e.stopPropagation(); setIsImageOpen(false); }}
     onTouchEnd={(e) => { e.stopPropagation(); e.preventDefault(); setIsImageOpen(false); }}
     className="absolute top-6 right-6 p-4 bg-white/10 hover:bg-white/20 rounded-full transition-colors z-10"
   >
     <X size={24} className="text-white" />
   </button>
   ```

3. **Same treatment for navigation arrows** (left/right chevrons, lines 718-736) — they already have `e.stopPropagation()` in their `onClick`, but add `onTouchEnd` handlers too for consistency on iOS:
   - Left arrow (line 719): add `onTouchEnd={(e) => { e.stopPropagation(); e.preventDefault(); setCurrentPhotoIndex(prev => prev > 0 ? prev - 1 : spot.image_urls!.length - 1); }}`
   - Right arrow (line 728): add `onTouchEnd={(e) => { e.stopPropagation(); e.preventDefault(); setCurrentPhotoIndex(prev => prev < spot.image_urls!.length - 1 ? prev + 1 : 0); }}`

4. **Add `onTouchEnd` to the backdrop motion.div** (line 703) for consistent dismiss on iOS:
   ```tsx
   <motion.div
     ...
     onClick={() => setIsImageOpen(false)}
     onTouchEnd={(e) => { if (e.target === e.currentTarget) { e.preventDefault(); setIsImageOpen(false); } }}
     ...
   >
   ```

5. **Increase X button touch target** — it already has `p-4` which is good (48px+), but ensure minimum 44x44px tap target. Add `min-w-[44px] min-h-[44px]` classes to be explicit.
