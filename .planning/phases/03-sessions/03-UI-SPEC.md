## Spacing Scale

All values are multiples of 4. Sourced from Tailwind v4 defaults as used across existing components.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps (`gap-1`), star icon spacing |
| sm | 8px | Inline element spacing (`gap-2`), button gaps |
| md | 16px | Card internal padding (`p-4`), form field gaps |
| lg | 24px | Section padding (`p-6`), tab content padding-top |
| xl | 32px | Not directly used in card views |
| 2xl | 48px | Not directly used in card views |
| 3xl | 64px | Not used in this phase |

**Exceptions:**
- Touch targets (Join/Leave/Cancel buttons): minimum 44px height to meet iOS HIG — use `min-h-[44px]` on all tappable action buttons.
- Avatar size in SessionCard: 32px (`w-8 h-8`) — exact match to ReviewList author avatar.
- Tab bar badge (participant count): use existing `px-1.5 py-0.5 rounded-full` pill pattern from reviews tab count badge.

## Component Inventory

New components for this phase. All mirror Phase 2 patterns exactly.

### SessionCard

**Purpose:** Displays a single upcoming session in SessionList.
**Layout:** Identical structure to ReviewList item — author row (avatar 32px + name + date) → session time row → note (optional) → action row.
**Card shell:** `bg-white rounded-xl p-4 border border-slate-100` — same as ReviewList item.
**Entrance animation:** `motion.div` with `initial={{ opacity: 0, y: 10 }}`, `animate={{ opacity: 1, y: 0 }}`, `transition={{ delay: index * 0.05 }}` — same as ReviewList.

**Author row:**
- Avatar: `w-8 h-8 rounded-full object-cover` — same as ReviewList
- Display name: `text-sm font-bold text-slate-800 truncate`
- Creator badge (own session): `text-[10px] font-bold text-sky-600 bg-sky-50 px-2 py-0.5 rounded-full uppercase` — label "Votre session"
- Date created: `text-xs text-slate-400`

**Session time row:**
- Calendar icon (lucide Calendar, size 14) in `text-sky-500`
- Date: `toLocaleDateString()` — locale-appropriate, no library
- Time: `toLocaleTimeString([], { hour: '2-digit', minute: '2-digit' })`
- Together: `text-sm font-semibold text-slate-800`

**Participant count:**
- Users icon (lucide Users, size 14) in `text-slate-400`
- Count: `text-xs text-slate-500`
- Format: `{count} participant{count !== 1 ? 's' : ''}`

**Session note (optional):**
- `text-sm text-slate-600 mt-1` — same as review comment text

**Action row (mutually exclusive states):**

| State | Condition | Button |
|-------|-----------|--------|
| Join | `!user_is_attending && creator_id !== user.id` | Outline button: `border border-sky-200 text-sky-600 hover:bg-sky-50 rounded-xl py-2 text-sm font-semibold min-h-[44px]` |
| Leave | `user_is_attending && creator_id !== user.id` | Outline button: `border border-slate-200 text-slate-500 hover:bg-slate-50 rounded-xl py-2 text-sm font-semibold min-h-[44px]` |
| Cancel session | `creator_id === user.id` | Outline button: `border border-red-200 text-red-500 hover:bg-red-50 rounded-xl py-2 text-sm font-semibold min-h-[44px]` |
| Not logged in | `!user` | No action button shown; session is read-only |

**Loading/disabled state:** `opacity-50 pointer-events-none` on button during async operation.

### SessionForm

**Purpose:** Inline form to create a new session on a spot. Shown only to authenticated users.
**Card shell:** `bg-white rounded-xl p-4 border border-slate-100` — same as ReviewForm.
**Section label:** `text-xs font-semibold text-slate-500 uppercase tracking-wide mb-2` — "Annoncer une session" — same as "Votre Avis" label in ReviewForm.

**Validation error:** `text-red-500 text-sm mt-2` — same as ReviewForm error display.

## Interaction States

### Join/Leave (optimistic)

On tap:
1. Button immediately switches state (Join → Leave or Leave → Join) — `user_is_attending` flips optimistically
2. Participant count increments/decrements immediately
3. Async DB call fires
4. On error: rollback both count and attending state, show inline error `text-red-500 text-sm` below card

### Cancel Session (destructive, requires confirmation)

On tap "Annuler la session":
1. Show inline confirmation row within the card (not a modal) — matches existing review delete inline UX
2. Confirmation row: `text-sm text-slate-600` question + two buttons side-by-side
   - Confirm: `text-red-500 border-red-200 rounded-xl py-1.5 px-4 text-sm font-semibold hover:bg-red-50`
   - Dismiss: `text-slate-500 border-slate-200 rounded-xl py-1.5 px-4 text-sm font-semibold hover:bg-slate-50`
3. On confirm: update `is_cancelled = true`, remove card from list with `AnimatePresence` exit animation `opacity: 0, y: -10`

### Tab Switching

Tap Sessions tab → reset stale sessions synchronously → trigger fetch → show spinner → render list.
No animation on tab content switch (current pattern in SpotDetail uses no animation for tab panels).

### Date Input

`<input type="datetime-local">` renders native OS picker on iOS/Android (Capacitor WebView).
On desktop (development): standard browser datetime-local widget.
Styled with Tailwind border/ring classes to match form aesthetic even though native picker overrides internal rendering on mobile.
