**3. Create `src/context/NotificationsContext.tsx`:**

This context provides:
- `ensurePushToken(userId: string): Promise<void>` — checks if token already in `push_tokens` for this user; if not, requests permission via `FirebaseMessaging.requestPermissions()`, gets token via `FirebaseMessaging.getToken()`, upserts to `push_tokens` with `{ user_id, token, platform: 'ios' }` and `onConflict: 'user_id,token'`. If permission denied or token empty, returns silently (never throws, never blocks caller).
- `permissionStatus: 'unknown' | 'granted' | 'denied' | 'loading'` — state updated on mount via `FirebaseMessaging.checkPermissions()` and after each `ensurePushToken` call.
- `checkPermission(): Promise<void>` — re-checks current permission state (used by Profile on mount).
- `hasToken: boolean` — whether the current user already has a token in `push_tokens`. Used by SessionForm and SessionCard to decide whether to show the inline banner.

Implementation details:
```typescript
import { createContext, useContext, useState, useEffect, useCallback, type ReactNode } from 'react';
import { FirebaseMessaging } from '@capacitor-firebase/messaging';
import { supabase } from '../lib/supabase';
import { useAuth } from './AuthContext';

type PermissionStatus = 'unknown' | 'granted' | 'denied' | 'loading';

interface NotificationsContextType {
  permissionStatus: PermissionStatus;
  hasToken: boolean;
  ensurePushToken: () => Promise<void>;
  checkPermission: () => Promise<void>;
}

const NotificationsContext = createContext<NotificationsContextType | undefined>(undefined);
```

In `NotificationsProvider`:
- On mount, call `checkPermission()` which sets `permissionStatus` to `'loading'`, then calls `FirebaseMessaging.checkPermissions()`, maps `receive` value to 'granted'/'denied'/'unknown'.
- On mount (if user exists), check if `push_tokens` has a row for user — sets `hasToken` state.
- Set up `FirebaseMessaging.addListener('tokenReceived', callback)` for token rotation — upserts new token to `push_tokens` silently. Cleanup listener on unmount.
- `ensurePushToken`:
  1. If no `user` from `useAuth()`, return.
  2. Query `supabase.from('push_tokens').select('token').eq('user_id', user.id).limit(1)`.
  3. If row exists, set `hasToken` to `true`, return early (already registered).
  4. Call `FirebaseMessaging.requestPermissions()`.
  5. If `receive !== 'granted'`, set `permissionStatus` to `'denied'`, return.
  6. Call `FirebaseMessaging.getToken()`.
  7. If no token, return.
  8. Upsert: `supabase.from('push_tokens').upsert({ user_id: user.id, token, platform: 'ios' }, { onConflict: 'user_id,token' })`.
  9. Set `permissionStatus` to `'granted'` and `hasToken` to `true`.
  10. Wrap entire function in try/catch — on error, `console.error('[Notifications]', error)` and return silently.

Export `NotificationsProvider` and `useNotifications` hook (with standard context-undefined error guard).
