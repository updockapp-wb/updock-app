**CONTEXT DEVIATION NOTE:** CONTEXT.md specifies "GROUP BY country sur table spots" for the spots-by-country KPI. However, RESEARCH.md (Critical Finding) confirmed that **no `country` column exists** in the spots table, and static spots in `spots.ts` also have no country field. Adding a DB column + migration + backfill would be significantly more complex and would still not cover static spots. Therefore, this plan derives country client-side from lat/lng coordinates using a bounding-box lookup, as recommended by RESEARCH.md. This supersedes the CONTEXT.md "GROUP BY country" approach. The end result is identical from the user's perspective: a sorted list of countries with spot counts.

`countryCodeToFlag(code: string): string` -- Converts ISO 3166-1 alpha-2 code to flag emoji:
```typescript
export function countryCodeToFlag(code: string): string {
  return code
    .toUpperCase()
    .split('')
    .map(c => String.fromCodePoint(0x1F1E6 + c.charCodeAt(0) - 65))
    .join('');
}
```
